Minimax

Minimax是一个非常著名的算法，常用于双人的博弈游戏，游戏双方一方为Max另一方为Min，Max希望能最大化自己的利益而Min希望最小化Min的利益。在开始学习Minimax之前，先看看别的一些相关的算法。

Maximax

Maximax的原则是使玩家总是选择可能受益最大的选择，而不考虑其他。让我们假设有这样一个游戏，两个玩家，甲和乙，各有一张红牌和蓝牌，双方选择一张同时展示。规则：如果甲选择红牌，无论乙选择什么，双方不输不赢；甲选择蓝牌，当乙选择了蓝牌，则甲赚1000元，乙选择红牌，甲输给乙1000元。可由下表展示甲的支付矩阵。

乙 乙
红 蓝
甲 红 0 0
甲 蓝 -1000 1000

依据Maximax算法的原则，甲始终会选择蓝牌，因为只有选择蓝牌才有可能获得最多的1000块收益。但是由上图也可以看出，当乙足够理智时，乙不会选择蓝牌，只会选择红牌，因为乙选择蓝牌只可能平或者输，选红牌则是平或赢。在这种情况下，甲如果按照Maximax算法，只会一直输下去。

Maximax算法本身是不合理的，它只考虑自己的最大获利情况而忽视了别等玩家的选择，naive。

Maximin

与Maximax相对的一个算法原则就是Maximin，与Maximax的乐观态度不同，Maximin是悲观的，总是考虑最悲观的情况。一个典型的情况就是老虎机，玩家有两个选择，玩或不玩，玩则消费1元，但可能中奖1000元，不玩则没有中奖机会当然也不会消费自己的1元。

机会 机会
赢 输
玩家 玩 1000 -1
玩家 不玩 0 0

由于玩家遵循Maximin的原则，他永远都不会玩老虎机，为了保证自己不会输1元。这种原则是相当消极的，不论回报有多高，他也不会冒任何风险。

单人游戏的Minimax

Minimax后悔原则是以Minimax原则为依据的，它更倾向于单人的游戏决策。它依赖于遗憾矩阵。想象有一家公司，正在决定是否需要推进一个研究项目，该项目回花费成本c元，如果项目成功，会带来r元的回报，失败则一无所有。

研究 研究
成功 失败
公司 推进 r-c -c
公司 不推进 0 0

依据Maximax原则，只要r-c大于零，公司就会一直推进项目；而依据Maximin原则，当有-c的风险时，公司永远不会推进项目。而依据Minimax，情况则会复杂一点。

我们需要绘制一个后悔矩阵，来展示机会成本，就拿这个公司来说，当公司推进了项目而且项目失败了，公司的后悔值是c，也就是项目成本；如果公司推进并成功了，将没有后悔值；如果公司没有推进但项目如果推进了就会成功，那后悔值是r-c即项目等回报；如果公司没推进且项目推进了也不会成功，则后悔值是0。

研究 研究
成功 失败
公司 推进 0 c
公司 不推进 r-c 0

我们的目的事最小化后悔值可能的最大值。即当r-c>c时，公司要推进项目，反之则不推进。换句话讲，只有当回报r大于两倍多成本c时才会推进项目。

双人游戏的Minimax

有这么一个例子：1943年，盟军收到报告称日军有一个护卫队将要渡海去加强他们的部队。护卫队有两条路可以走，一条往北，一条往南。盟军必须决定如何调遣他们的侦察机——往南还是往北——为了尽快拦截护卫队。下面的矩阵显示了日本和盟军可能作出的决定和盟军可能的轰炸天数。

日军路线 日军路线
北 南
盟军路线 北 2 2
盟军路线 南 1 3

根据这个矩阵可以看出，盟军最好的情况是盟军往南，日军往南，可以轰炸3天，而日军最好的情况时日军往北，盟军往南，盟军只能轰炸1天。

为了最小化最糟糕的情况，盟军应该选择向北侦查，这可以让他们确保有2天的轰炸时间，而往南则要面临只能轰炸1天的风险。所以，根据minimax的原则，盟军要选择北方。

日军可以采用相同的策略。如果往南，日军要面临3天轰炸的风险，所以日军也选择北方。

事实上，盟军和日军都选择了北方，日军受到了2天的轰炸。

根据之前的矩阵，日军和盟军双方都确定都选择北方是对双方都是最好的结果。在对手都是理性的情况下，不可能有更好的结果了。在这个例子中，minimax和maximin产生了相同的结果。如果盟军选择maximax，他们会选择南方，而结果是只有1天的轰炸时间。
